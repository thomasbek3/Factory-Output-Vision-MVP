# Implementation Docs

Use this folder for execution planning, not product vision.

## Structure

Use one dated folder per planning snapshot:

`docs/IMPLEMENTATION/YYYY-MM-DD/`

Recommended files inside each dated folder:
- `README.md`
- `PHASE_PLAN.md`
- `BACKEND_TASKS.md`
- `FRONTEND_TASKS.md`
- `CHECKLIST.md`
- `RISKS.md`

## Naming Rules

- always use ISO dates: `YYYY-MM-DD`
- avoid spaces and slashes in folder names
- keep one folder per major planning pass
- update the newest folder instead of spreading active work across many places
